In the exercise instructions, I asked you to think a bit about improving the design of the program. Here are a few more things to think about:

  • MancalaBoard defines a constant, BOARDSIZE to represent the number of pits in the board for each player. Is this fully exploited? That is, are there any parts of the code that should use this value but don't?
  • Are there any other constants we should declare? (I generally ask, "are there any magic numbers in the code?")
  • You've done some reading about encapsulation, though we haven't discussed it yet. How could we improve the encapsulation of this programs and its classes?
  • This discussion of Kalah suggests a different way of thinking about game state (look at the second gray box with "North" and "South" at top and bottom. How would your implementation (both data and methods) change if you followed this implementation? Would it make your code simpler or more complex?
